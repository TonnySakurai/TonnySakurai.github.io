Python学习记录之数据类型

Tonny Liu Sep 02, 2018
  • Data Type
  • Python
  •  Views
  • Share to Weibo
  • Share to Facebook
  • Share to QQ

在使用底层语言C或者C++时，我们有很大一部分工作都集中于用对象去表现应用领域的组件。在这一过程中我们需要部署内存结构、管理内存分配、实现搜索和读取例程等。这些工作乏味且容易出错，并且往往背离程序的真正目标。
而在Python中这些令人头疼的大部分工作都消失了，因为Python提供了强大的数据类型作为语言的组成部分，在你开始解决问题之前，往往没必要编写对象的实现，并且内置的数据类型往往比定制的数据结构更有效率。

Python的核心数据类型包括：数字，字符串，列表，字典，元组，文件，集合等等。本文将分别介绍这些数据类型的特点及使用方法（数字这一类型因为与其它语言差别不大，故不在此说明）。在接下来的说明中我们将看到，Python中数据类型与C和C++显著不同的一点是：Python中没有类型声明，运行的表达式的语法决定了创建和使用的对象的类型。

数字

数字计算时混合类型自动升级

比如在 40 + 3.14 的情况下，计算结果会是浮点数。因为在混合类型的表达式中，Python 首先将被操作的对象转化成其中最复杂的操作对象的类型，然后再对相同类型的 操作对象进行数学运算。Python 是这样划分数字类型的复杂度的：整数比浮点数简单，浮点数比复数简单。

数字的比较

Python 允许我们把多个比较连续起来执行范围测试，如：

>>> x, y, z=2, 4, 6
>>> x < y < z
True
>>> x < y > z
False

这里 x < y < z 等同于布尔测试（x < y and y < z）

除法：传统除法、Floor除法和真除法

X/Y

传统除法和真除法。在 Python 2.6 或之前的版本，这个操作对于整数会省去小数部分，对于浮点数会保持小数部分。在python 3.0版本中将会变成真除法（无论任何类型都会保持小数部分）。

X//Y

Floor除法。在 Python 2.6 和Python 3.0 中均能使用。这个操作不考虑操作对象的类型，总会省略掉结果的小数部分，剩下最小的能整除的整数部分。注意，在 Python 3.0 中，//的结果的数据类型总是依赖于操作数的类型：如果操作数中有一个是浮点数，结果就是浮点数；否则，结果是一个整数。其次，floor 除法是把结果向下截断到它的下层，即真正的结果之下的最近的整数，其直接效果是向下舍入。如5 // -2 = -3。而真正的截断操作可以通过math.trunc来实现。

>>> 10 / 4.0
2.5

复数

复数也是 Python 的核心数字类型之一。复数表示为两个浮点数（实部和虚部）并在虚部增加了 j 或 J 的后缀。复数可以通过标准的cmath模块中的工具进行处理。下面是一些复数运算的例子。

>>> 1j * 1J
(-1 + 0j)
>>> 2 + 1j * 3
(2+3j)

十六进制、八进制和二进制记数

Python 整数能够以十六进制、八进制和二进制记数法来编写。
十六进制数以 0x 或 0X 开头，后面接十六进制的数字 0 ~ 9 和 A ~ F。十六进制数的数字编写成大写或小写都可以。
八进制常量以数字 0o 或 0O 开头（ 0 和小写或大写的字母 O ），后面接着数字 0 ~ 7 构成的字符串。
二进制以 0b 或者 0B 开头，后面跟着二进制数字（0 ~ 1）

>>> 0o1, 0o20, 0o377
(1, 16, 255)
>>> 0x01, 0x10, 0xFF
(1, 16, 255)
>>> 0b1, 0b10000, 0b11111111
(1, 16, 255)

这里Python默认地用十进制显示，但我们可以通过内置函数把整数转换为其他进制地数字字符串：
oct() 函数会将十进制转换为八进制数，hex() 函数会将十进制转换为十六进制数，而 bin() 会将十进制转换为二进制。

小数和分数

集合

字符串

字符串是一个有序的字符的集合，用来存储和表现基于文本的信息，它有一堆引号（单引号双引号都行）以及引号中间的信息组成。 在python中，字符串作为不可变的序列存在。由于是序列的一种，字符串支持各种有关于元素位置信息的操作：

获取字符串的长度

内置函数lan()可以获得字符串的长度：

>>> S = 'Spam'
>>> lan(S)
4

获取字符串其中某一个字符

S[a]代表相对于第一个字符偏移a位的字符，即:

>>> S[1]
'p'

反向索引

Python支持反向索引，即:

>>> S[-1]
'm'

分片

Python支持分片(slice)的操作，这是一种一步就能够提取整个分片的方法，即:

>>> S[1:3]
'pa'

分片的一般形式为X[I:J]，表示“取出在X中从偏移量为I，直到但不包括偏移量为J的内容”。在一个分片中，左边界默认为0，并且右边界默认为分片序列的长度，如以下例子所示：

>>> S[ :3] 
'Spa'    
>>> S[ :-1] 
'Spa'    
>>> S[ : ]
'Spam'

字符串的合并

作为一个序列，字符串也支持使用加号进行合并（将两个字符串合并成为一个新的字符串），或者重复。例如：

>>> S + 'xyz'
'Spamxyz'    
>>> S*2  
'SpamSpam'

字符串具有不可变性

字符串在python中具有不可变性——在创建之后不能就地改变。例如不能通过对其某一位置进行赋值而改变字符串，如下操作是非法的：

>>> S[0] = 'Z'

但可以通过建立一个新的字符串并以同一个变量名对其进行赋值以达成目的：

>>> S = 'Z' + S[1: ]
>>> S
'Zpam'

列表

序列操作：

由于列表是序列的一种，列表支持所有的我们对于字符串讨论过的序列操作，如：

>>> L = [123, 'spam', 1.23]
>>> len(L) 
3

>>> L[0]
123

>>> L[ :-1] 
[123, 'spam']

>>> L + [4, 5, 6] 
[123, 'spam', 1.23, 4, 5, 6]

类型特定的操作：

增加、减少列表的大小：

>>> L.append('NI')    #增加值为NI的一项
>>> L
[123, 'spam', 1.23, 'NI']

>>> L.pop(2)          #移除给定偏移量为2的一项
1.23
>>> L
[123, 'spam', 'NI']

边界检查：

超出列表末尾之外的索引或赋值会导致错误。

嵌套：

Python核心数据类型的一个优秀的特性就是它们支持任意的嵌套。能够以任意的组合对其进行嵌套，并可以多个层次进行嵌套（例如，能够让一个列表包含一个字典，并在这个字典中包含另一个列表等）。这种特性的一个直接的应用就是实现矩阵。

>>> M = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]
>>> M[1]
[4, 5, 6]
>>> M[1][2]
6

注：这种矩阵结构适用于小规模的任务，但对于更重要的数值运算而言，可以考虑使用Python数值扩展包，如NumPy系统。

列表解析（list comprehension expression）：

列表解析源自集合的概念。它是一种通过对序列中的每一项运行一个表达式的方式来创建一个新的列表的方法。如下面各例所示：

>>> col2 = [row[1] for row in M]       #把矩阵M的每个row中的row[1]，放在一个新的列表中
>>> col2 
[2, 5, 8]

>>> [row[1] + 1 for row in M]
[3, 6, 9]

>>> [row[1] for row in M if row[1] % 2 == 0]
[2, 8]

>>> diag = [M[i][i] for i in [0, 1, 2] ]
[1, 5, 9]

>>> doubles = [c * 2 for c in 'spam']
>>> doubles
['ss', 'pp', 'aa', 'mm']

字典

Python中的字典是完全不同的东西：它们不是序列，而是一种映射（mapping）。映射是一个其他对象的集合，但是它们是通过键而不是相对位置来存储的。

作为常量编写时，字典编写在大括号中，并包含一系列的“键：值”对。如下所示：

>>> D = {'food' : 'Spam', 'quantity' : 4, 'color' : 'pink'}

索引

我们可以通过键对这个字典进行索引来读取或改变键所关联的值。字典的索引操作使用的是和序列相同的语法，但在方括号中的元素是键，而不是相对位置。

>>> D['food']
'Spam'

>>> D['quantity'] += 1
>>> D
{'food': 'Spam', 'quantity' : 5, 'color' : 'pink'}

赋值

可以从一个空的字典开始，然后每次以一个键来填写它。与列表中禁止边界外的赋值不同，对一个新的字典的赋值会创建该键：

>>> D = {}
>>> D['name'] = 'Bob'
>>> D['job'] = 'dev'
>>> D['age'] = 40
>>> D
('age' : 40, 'job' : 'dev', 'name' : 'Bob' )

通过键索引一个字典往往是Python中编写搜索的最快方法。

重访嵌套

如果需要用字典去描述的事物较复杂，需要用到具有嵌套结构的字典。如下所示：

>>> rec = {'name' : {'first' : 'Bob', 'last' : 'Smith'}},
            'job' : ['dev', 'mgr'],
            'ago' : 40.5}

在这里，在顶层再次使用了三个键的字典，但是值的情况变得复杂得多：一个嵌套的字典作为name的值，并用一个嵌套的列表作为job的值。需要进行索引时，首先需要索引到所需要的值对应的键，之后根据该键对应的数据类型进行索引。

>>> rec['name']
{'first' : 'Bob', 'last' : 'Smith'}

>>> rec['name']['last']
'Smith'

>>> rec['job']
['dev', 'mgr']
>>> rec['job'][-1]
'mgr'

判断是否存在

In关系表达式允许我们查询字典中一个键是否存在。

>>> 'f' in D
False

元组

元组对象（tuple）基本上就像一个不可以改变的列表。就像列表一样，元组是序列，但是它具有不可变性，和字符串类似。从语法上讲，它们编写在圆括号中，它支持任意类型、任意嵌套以及常见的序列操作。

>>> T = (1, 2, 3, 4)
>>> len(T)
4

>>> T + (5, 6)
(1, 2, 3, 4, 5, 6)

但下述操作是非法的：

>>> T[0] = 2
error text
>>> T.append(4)
error

为什么要用元组?

元组在实际中往往并不像列表这样常用，但是它的关键是不可变性。如果程序中以列表的形式传递一个对象的集合，它可能在任何地方改变；如果使用元组的话，则不能。

文件

文件对象是Python代码对电脑上外接文件的主要接口。要创建一个文件对象，需调用内置的open函数以字符串的形式传递给它一个外部的文件名以及一个处理模式的字符串。比如：以‘w’的模式创建一个文本输出文件：

>>> f = open('data.txt', 'w')
>>> f.write('hello\n')
6                                             #返回一个所写内容的大小
>>> f.write('world'\n)
6
>>> f.close()

这样就在当前文件夹下创建了一个文件，并向它写入文本（如果需要读取电脑上其他位置的文件，文件名也可以是完整的路径）
